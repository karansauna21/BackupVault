# Security Policy

We take the security of BackupVault seriously. Cryptographic assurance and system integrity are core to the application's design.

---

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 1.0.x   | Yes       |
| < 1.0   | No        |

---

## Reporting a Vulnerability

**Do NOT report security vulnerabilities via public GitHub issues.**

If you discover a security vulnerability in BackupVault, please report it privately through one of the following channels:
- **Email**: security@backupvault.org
- **Direct Submission**: Encrypted report using our PGP Key: `0xBE5F...` (Refer to public keyservers).

### What to Include in a Security Report
Please provide:
1. A detailed description of the vulnerability.
2. A Proof of Concept (PoC) showing how the vulnerability can be triggered.
3. Steps we can take to reproduce the issue locally.
4. Estimation of potential impact (e.g. data leak, local privilege escalation, encryption key bypass).

### Our Response Timeline
- **Triage**: Within 48 hours.
- **Vulnerability Patch**: Within 14 business days (depending on severity).
- **Public Disclosure**: Coordinated disclosure after patches have been published.
